# SOP: การจัดทำ BOQ งานท่อร้อยสายสื่อสารใต้ดินผ่านระบบ Conduit BOQ

**Purpose:** ใช้เป็นคู่มือมาตรฐานสำหรับถ่ายทอดความรู้และปฏิบัติงาน  
**Audience:** พนักงานผู้จัดทำ BOQ, ผู้ตรวจสอบ, ผู้ดูแลระบบ, KM Agent  

---

## 1. วัตถุประสงค์ของ SOP

- ทำให้การจัดทำ BOQ ใช้ขั้นตอนเดียวกัน
- ลดการใช้ไฟล์และสูตรเฉพาะบุคคล
- ใช้ price list กลางและ calculation logic กลาง
- ทำให้ BOQ ตรวจสอบย้อนกลับได้
- ใช้เป็นเอกสารถ่ายทอดความรู้ให้บุคลากรใหม่

---

## 2. ขอบเขต

ครอบคลุม:

- Login และตรวจสอบสิทธิ์
- สร้าง BOQ draft
- เพิ่ม route/พื้นที่ก่อสร้าง
- ค้นหาและเพิ่มรายการจาก price list
- กรอกปริมาณและตรวจยอด
- ตรวจ Factor F และ VAT
- บันทึก BOQ
- Print และ Excel export

ไม่ครอบคลุม:

- งานจัดซื้อจัดจ้าง
- การอนุมัติเต็มรูปแบบ
- การบริหาร stock วัสดุ
- การปฏิบัติงานภาคสนาม

---

## 3. ขั้นตอนปฏิบัติงาน

### Step 1: เข้าสู่ระบบ

1. เปิดระบบ Conduit BOQ
2. Login ด้วยบัญชีที่ได้รับอนุญาต
3. ตรวจสอบว่า user profile มีสถานะและ role ที่ถูกต้อง

ผลลัพธ์ที่ต้องได้:

- ระบบแสดง dashboard ตามสิทธิ์ผู้ใช้
- ผู้ใช้เห็นเมนูสร้าง BOQ หากมีสิทธิ์

### Step 2: สร้าง BOQ

1. ไปที่หน้า Create BOQ
2. กรอกข้อมูลโครงการ เช่น ชื่อโครงการ ผู้ประมาณราคา วันที่ และข้อมูลที่เกี่ยวข้อง
3. บันทึกเป็น draft

ผลลัพธ์ที่ต้องได้:

- ระบบสร้าง BOQ header
- BOQ มี owner และ org/department/sector ตามผู้สร้าง

### Step 3: เพิ่ม Route

1. เปิดหน้า edit ของ BOQ
2. เพิ่มเส้นทางหรือพื้นที่ก่อสร้าง
3. ตั้งชื่อ route ให้สื่อความหมาย
4. ระบุ construction area หากมี

แนวปฏิบัติ:

- แยก route ตามพื้นที่ก่อสร้างจริง
- ตั้งชื่อ route ให้ตรวจสอบย้อนกลับได้
- หลีกเลี่ยงการรวมหลายพื้นที่ที่ควรแยกต้นทุน

### Step 4: เพิ่มรายการ BOQ

1. เลือก route ที่ต้องการ
2. ค้นหารายการจาก price list
3. เลือกรายการที่ตรงกับงานจริง
4. ระบุ quantity

ข้อควรตรวจ:

- item name ตรงกับงานจริง
- unit ถูกต้อง
- quantity สมเหตุสมผล
- ราคามาจาก price list กลาง

### Step 5: ตรวจยอดคำนวณ

ระบบคำนวณ:

- material cost
- labor cost
- unit cost
- item total
- route total
- BOQ grand total
- Factor F
- VAT 7%
- final total

ผู้ใช้ควรตรวจ:

- route total เทียบกับรายการใน route
- BOQ grand total เทียบกับทุก route
- Factor F แสดงผลถูกต้อง
- total with VAT พร้อมใช้งาน

### Step 6: บันทึก

1. กด Save
2. รอระบบบันทึก BOQ, routes, items และ snapshot
3. ตรวจว่าไม่มี error

ข้อควรระวัง:

- ห้ามออกจากหน้าก่อน save สำเร็จ
- หากมี error ให้บันทึก screenshot และแจ้งผู้ดูแลระบบ

### Step 7: Print / Export

1. เปิดหน้า print
2. ตรวจรายละเอียด BOQ
3. ตรวจ route detail, summary และ Factor F supplement
4. Print หรือ export Excel ตามความต้องการ

ผลลัพธ์:

- เอกสาร BOQ ที่ใช้ข้อมูลเดียวกับฐานข้อมูล
- ลดความเสี่ยงจากการจัดรูปแบบ/แก้ตัวเลขนอกระบบ

## 5. Common Issues and Handling

| Issue | วิธีจัดการ |
|---|---|
| หา item ไม่เจอ | ตรวจคำค้น/category และบันทึกเป็น catalog gap |
| ยอดรวมไม่ตรง | ตรวจ quantity, route items และแจ้งผู้ดูแลหากยัง mismatch |
| Factor F ไม่แสดง | ตรวจ reference table หรือแจ้งผู้ดูแลระบบ |
| ไม่มีสิทธิ์แก้ BOQ | ตรวจ role/status/owner กับ admin |
| Export ผิดพลาด | ลอง refresh และแจ้ง error พร้อม BOQ id |
